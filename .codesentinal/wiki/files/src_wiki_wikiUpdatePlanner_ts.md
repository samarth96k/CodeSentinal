# Purpose
To serve as the automated engine for the LLM Wiki system, specifically responsible for the generation, maintenance, and lifecycle management of repository knowledge.

# Responsibilities
*   Generating repository memory files to facilitate future codebase understanding.
*   Managing the lifecycle of wiki documentation within the system.
*   Orchestrating the planning process for repository knowledge updates.

# Architectural Role
Acts as the Repository Knowledge Layer, bridging the gap between raw codebase intelligence and structured, LLM-accessible documentation.

# Critical Review Context
When reviewing PRs affecting this file, focus on:
*   **Knowledge Consistency:** Ensure the planner maintains a unified structure for all generated wiki content.
*   **Context Quality:** Verify that the output generated by this planner is actionable and relevant for subsequent LLM operations.
*   **Risk Mitigation:** Ensure that the parsing logic—specifically for JSON—includes robust error handling to prevent system crashes or data corruption from malformed inputs.

# Related Components
*   `src/config/runtimeConfig.js`: Provides necessary environment or system-wide configuration settings.
*   `src/llm.js`: The underlying engine used for natural language processing and content generation.
*   `src/wiki/wikiPathMapper.js`: Manages the mapping and location logic for wiki assets.
*   `src/wiki/wikiUpdatePrompt.js`: Contains the prompt templates and logic required for content generation.

# Repository Memory
The `wikiUpdatePlanner.ts` is the central coordinator for the wiki system. It relies on internal logic to prepare instructions for the LLM. A primary concern for future maintenance is the reliability of the JSON parsing process; any changes to the output format of this planner or its dependencies must be verified against the parsing safety requirements to ensure the system remains resilient to invalid data.
